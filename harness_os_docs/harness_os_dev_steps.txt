阶段 1：Project Manager 核心
====================================================================

  参考文档：06_TASK_DECISION_PROJECT_MANAGER.md §6
  验收标准：11_ACCEPTANCE_CRITERIA.md §5

  1.1 实现 create 流程 (src/project/create.ts)
      - 创建项目目录
      - 初始化 Git repo
      - 注入 AGENTS.md 模板
      - 创建 .project/ 结构
      - 写入 manifest.json
      - 扫描技术栈（tech-stack.md）
      - 生成 repository-map.md
      - 创建 checkpoint
      - 生成 project-created 报告

  1.2 实现 open 流程 (src/project/open.ts)
      - 校验路径和 Git repo
      - 检查 AGENTS.md 是否存在
      - 检查 .project/ 完整性
      - 读取并校验 manifest.json
      - 刷新 repository-map.md
      - 读取 git status
      - 输出 project open summary

  1.3 实现 init/repair 流程 (src/project/repair.ts)
      - 创建缺失目录
      - 补齐 manifest 字段
      - 刷新 tech-stack.md / repository-map.md
      - 检查 AGENTS.md 必填章节
      - 生成 repair report

  1.4 CLI 集成
      - harness create <name>
      - harness open <path>
      - harness init / harness repair

阶段 3：Task Manager 基础生命周期
====================================================================

  参考文档：06_TASK_DECISION_PROJECT_MANAGER.md §7
  验收标准：11_ACCEPTANCE_CRITERIA.md §7

  3.1 Task Record 生成 (src/task/create.ts)
      - 生成 task id
      - 规范化任务标题
      - 推断 task type
      - 提取显式文件、命令
      - 写入 .project/tasks/active/<task-id>.md
      - 写入 .project/tasks/active/<task-id>.json

  3.2 Task 状态机 (src/task/state-machine.ts)
      - 状态流转：created → ready → running → blocked → paused
        → verifying → completed / failed
      - 非法状态转移报错

  3.3 Task 完成流程 (src/task/complete.ts)
      - 收集 changed files
      - 生成 task summary
      - 移动 active → completed 或 failed
      - 生成 completion event

  3.4 Task Manager API
      - create / start / pause / resume / block / complete / fail
      - linkRun / linkContext / linkCheckpoint / linkDecision

阶段 4：Context Engineering 核心
====================================================================

  参考文档：05_CONTEXT_ENGINEERING.md
  验收标准：11_ACCEPTANCE_CRITERIA.md §9

  4.1 Source Collectors
      - AGENTS.md Collector — 提取核心规则
      - Project State Collector — 读取 project.md, manifest.json
      - Task Collector — 读取当前 task
      - Git Collector — 读取 git status/diff/log
      - Decision Collector — 读取相关 ADR

  4.2 Relevance Engine (src/context/relevance.ts)
      - 显式文件匹配（优先）
      - Git diff 文件匹配
      - 测试文件映射
      - 关键词匹配（ripgrep）

  4.3 Context Pack 生成 (src/context/build.ts)
      - 组装 Context Pack JSON
      - 组装 Context Pack Markdown
      - 写入 .project/context/<run-id>.json
      - 写入 .project/context/<run-id>.md

  4.4 Context Budget Manager (src/context/budget.ts)
      - Token 估算（char/4）
      - P0 → P3 优先级裁切
      - P0 保留规则

  4.5 Context Snapshot & Refresh
      - 保存每次 run 后的 Context 快照
      - 文件变更时触发 refresh

阶段 5：Governance 基础层
====================================================================

  参考文档：08_GOVERNANCE_SECURITY.md
  验收标准：11_ACCEPTANCE_CRITERIA.md §11

  5.1 Policy Engine (src/governance/policy.ts)
      - 策略来源：Global → Project Manifest → AGENTS.md
      - 策略检查流程：Operation + Risk → Allow/Deny/Requires-Approval
      - 路径保护：workspace boundary check
      - 命令风险分类：危险命令检测

  5.2 Approval Gate (src/governance/approval.ts)
      - 审批请求创建
      - 审批状态管理（pending / approved / denied）
      - CLI 交互式提示
      - 非交互模式处理

  5.3 Secret Redactor (src/governance/redactor.ts)
      - 常见 secret 模式检测（API keys, tokens, .env）
      - 统一替换为 [REDACTED]
      - 在 Skill 输出和日志中自动脱敏

  5.4 Context Pack 治理集成
      - Context Pack 中包含审批规则
      - 可用 Skills 列表

阶段 6：Verification Pipeline
====================================================================

  参考文档：09_VERIFICATION_OBSERVABILITY.md §6-10
  验收标准：11_ACCEPTANCE_CRITERIA.md §12

  6.1 命令检测器 (src/verification/commands.ts)
      - 从 AGENTS.md 读取命令
      - 从 manifest.json 读取命令
      - 从 package.json 推断命令
      - 标记不确定性（inferred 命令标 uncertain）

  6.2 Verification Plan 生成 (src/verification/plan.ts)
      - 按优先级排序：lint → typecheck → test → build
      - 标记 required / optional
      - 设置 timeout

  6.3 Verification Runner (src/verification/runner.ts)
      - 通过 Shell Skill 执行命令
      - 捕获 exit code / stdout / stderr
      - 汇总结果（passed / failed / partial / skipped）

  6.4 Verification Report (src/verification/report.ts)
      - 写入 .project/reports/verification/<run-id>.md
      - 包含命令结果表、风险说明
